--- layout: resource title: Province intro: "Shopowners can specify which countries they will ship to (in the Regions & Taxes page, located in the Preferences menu), and these countries are made available through the API using the country resource. If any of those countries have states or provinces, those states or provinces are also registered as shipping destinations, each of which can have its own state or provincial sales tax. As with countries, you can modify the sales tax for each state or province to account for surtaxes or exemptions that apply to the shop." ---

Among the sets of states and provinces supported by Shopify are:

  • Australia's 8 states and territories
  • Canada's 13 provinces and territories
  • Mexico's 32 states
  • Spain's 52 provinces
  • United States' 62 states and territories

States and provinces are represented by the province resource; we use the term "province" to cover both.

Special note for U.S. Shops using the County Tax feature

County Tax (currently in beta) is a new feature that automatically calculates the state and county sales taxes for a purchase based on the customer's zip code. If the shop has County Tax enabled (new shops have it enabled by default, and as of April 2012, nearly half of all active shops do), the tax rates defined for the state will be ignored in favor of the tax calculated by County Tax, which is more accurate.

What can you do with provinces using the Shopify API?

The Shopify API lets you do the following with provinces:

  • Get information about one or more states or provinces of a country currently registered with the shop.
  • Count all the states or provinces of a country currently registered with the shop.
  • Create a new province
  • Modify the sales tax of a state or province of a country already registered with the shop.

Province properties

Name Description
code The standard abbrevation for the state or province. The code for a given state or province will be the same as the code for the same state or province in another shop.
id The unique numeric identifier for the state or province. Note that the id for a given state or province in one shop will not be the same as the id for the same state or province in another shop.
name The full name of the state or province.
tax The national sales tax rate to be applied to orders made by customers from that country.
tax_name The national sales tax rate to be applied to orders made by customers from that country.
tax_type The national sales tax rate to be applied to orders made by customers from that country.
tax_percentage The national sales tax rate to be applied to orders made by customers from that country.

Endpoints

GET /admin/countries/:country_id/provinces.json

Query parameters

By default, requesting a list of provinces returns all the provinces for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
country_id The id of a country. Use this parameter to restrict the returned list of provinces to only those belonging to a specific country.
fields A comma-separated list of the properties you want returned for each province in the result list. By default, the returned list of provinces is a list of provinces with all the properties included. Use this parameter to restrict the returned list of provinces to showing only those properties you specify, with each property name separated by a comma.
since_id The lower bound on a provinceâ€™s id. Use this parameter to restrict the returned list to only provinces whose id is greater than the specified id.

GET /admin/countries/:country_id/provinces/:province_id.json

Example: Get a single state or province registered with the shop

In this example, we want to get the properties of the province of Quebec, whose id in our example shop is 224293623.

Request:

GET /admin/countries/3378932/provinces/224293623.json

Response:

HTTP/1.1 200 OK[]
{
  "province": {
    "code": "QC",
    "id": 224293623,
    "name": "Quebec",
    "tax": 0.09,
    "tax_name": null,
    "tax_type": null,
    "tax_percentage": 9.0
  }
}
GET /admin/countries/:country_id/provinces/count.json

Example: Count provinces or states of a country registered with the shop

In this example, we want to count the provinces of Canada, whose id in our example shop is 3378932.

Request:

GET /admin/countries/3378932/provinces/count.json

Response:

HTTP/1.1 200 OK[]
{
  "count": 13
}
PUT /admin/countries/:country_id/provinces/:province_id.json

Example: Modify the sales tax of a state or province registered with the shop

In this example, we want to change the sales tax of Quebec, a province of Canada, to 15%. In our example shop, Canada's id is 3378932 and Quebec's id is 224293623.

Request:

PUT /admin/countries/3378932/provinces/:224293623.json

Request body:

{
   "province": {
     "tax": 0.15
   }
}

Response:

HTTP/1.1 200 OK[]
{
  "province": {
    "code": "QC",
    "id": 224293623,
    "name": "Quebec",
    "tax": 0.15,
    "tax_name": null,
    "tax_type": null,
    "tax_percentage": 15.0
  }
}
