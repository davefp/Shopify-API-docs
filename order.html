What can you do with orders using the Shopify API?

The Shopify API lets you do the following with order:

  • Get information about one or more orders
  • Count of all or specific orders
  • Close an order that has been processed
  • Reopen a closed order
  • Cancel an order
  • Modify one of the following attributes of an order:
    • Customer's email address
    • Notes about the order
    • The attributes of the notes about the order
    • Whether or not the customer will accept email marketing sent to the email address s/he provided
  • Add a metafield to an existing order
  • Delete an existing order

You should note that orders can't be created either through the API or the shop's admin panel. The only way to create an order is to visit the shop, add items to the cart and check out.

You should also note that you can change only a few of an order's attributes using the API, nor can you change the items or the quantities in an order.

Query parameters

By default, requesting a list of orders returns all the orders for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
created_at_max The upper bound on the date and time of creation (its created_at value) for the orders you want to retrieve. Use this parameter to retrieve only orders that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the orders you want to retrieve. Use this parameter to retrieve only orders that were created after a given date and time.
fields A comma-separated list of the properties you want returned for each orders in the result list. By default, the returned list of orders is a list of orders with all the properties included. Use this parameter to restrict the returned list of orders to showing only those properties you specify, with each property name separated by a comma.
financial_status The financial status of the orders you want to retrieve. Valid values are:
  • abandoned: Show only orders that have been abandoned.
  • any: Show orders that are authorized, pending and paid. Note that this will not return abandoned orders.
  • authorized: Show only orders that have been authorized.
  • paid: Show only orders that have been paid.
  • pending: Show only orders that are pending.
  • refunded: Show only orders that have been refunded.
  • voided: Show only orders that have been voided.
fulfillment_status The fulfillment status of the orders you want to retrieve. Allowed values are:
  • any (default): Show orders with any fulfillment status.
  • partial: Show only orders that have been partilly shipped.
  • shipped: Show only orders that have been shipped.
  • unshipped: Show only orders that have not yet been shipped.
limit The maximum number of orders that should be returned. By default, the maximum number of orders returned is 50. Use this parameter to change the maximum number of returned orders from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the orders in a shop.
since_id The lower bound on the id for the orders you want to retrieve. Use this parameter to restrict the returned list to only orders whose ids are ''greater'' than a given id.
status The open/closed status of the orders you want to retrieve. Allowed values are:
  • any: Show both open and closed orders.
  • closed: Show only orders that have been closed.
  • open (default): Show only open orders.
updated_at_max The upper bound on the date and time of a ***'s last update (its updated_at value) for the ***s you want to retrieve. Use this parameter to retrieve only ***s that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a ***'s last update (its updated_at value) for the ***s you want to retrieve. Use this parameter to retrieve only ***s that were updated after a given date and time.

POST /admin/orders/:order_id/cancel.json

Query parameters

You can perform additional actions with a cancellation by including the following optional query parameters.

Name Description
amount The upper bound on the date and time of creation (its created_at value) for the orders you want to retrieve. Use this parameter to retrieve only orders that were created before a given date and time.
email The email address where the cancellation notice will be sent. If you use this parameter, an email notifying the receiver about the order being cancelled will be sent to this address.
reason The reason why the order was cancelled. Valid values are:
  • customer: The customer requested that the order be cancelled.
  • fraud: The order was made as part of a fraud attempt.
  • inventory: The order was cancelled because of a lack of inventory.
  • other (default): The order was cancelled for an unspecified reason or a reason that doesn't fit into any of the other categories in this list.
restock A boolean value (whose default is false) that if set to true, restocks the items from this order back into your shop's inventory.
