--- layout: resource title: Collect intro: "A collect is an object that connects a product to a custom collection. For every product membership in a custom collection (a product can be a member of more than one collection), there exists a collect that tracks the ids of both. Adding a product to a collection involves creating a collect; removing a product from a collection means deleting the corresponding collect." ---

Collects are for specifying the membership of products in custom collections only; smart collections use rules to determine which products are their members. Creating a collect that links a product to a smart collection results in a 403 (forbidden) error.

What can you do with collects using the Shopify API?

The Shopify API lets you do the following with collects:

  • Get information about one or more collects
  • Count all or specific collects
  • Create a new collect
  • Delete a collect

Unlike many Shopify resources, collects aren't apparent to shopowners; they're objects for managing the many-to-many relationship between products and custom collections.

collect properties

Name Description
collection_id The id of the custom collection containing the product.
created_at The date and time when the collect was created. The API returns this value in ISO 8601 format.
featured
id The unique numeric identifier for the collect.
position A number specifying the order in which the product appears in the custom collection, with 1 denoting the first item in the collection. This value applies only when the custom collection's sort-order property is set to manual.
product_id The id of the product contained by the custom collection.
updated_at The date and time when the collect was last modified. The API returns this value in ISO 8601 format.

Endpoints

GET /shops/:shop_id/collects.json

Query parameters

By default, requesting a list of collects returns all the collects for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
collection_id The id of a specific custom collection. Use this parameter to retrieve only those collects that refer that collection.
fields A comma-separated list of the properties you want returned for each collect in the result list. By default, the returned list of collects is a list of collects with all the properties included. Use this parameter to restrict the returned list of collects to showing only those properties you specify, with each property name separated by a comma.
product_id The id of a specific product. Use this parameter to retrieve only those collects that refer that product.

POST /admin/collects.json

Example: Create a new collect

In this example, we want to add the product whose id is 921728736 to the custom collection whose id is 841564295.

Request:

POST /admin/collects.json

Request body:

{
  "collect": {
    "product_id": 921728736,
    "collection_id": 841564295
  }
}

Response:

HTTP/1.1 201 Created[]

    "collection_id": 841564295,

  }
