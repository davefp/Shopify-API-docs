--- layout: resource title: Fulfillment intro: A fulfillment represents a shipment of one or more items in an order. When we say that an order has been completely fulfilled, we mean that all the items that make up that order have been sent to the customer. ---

Ideally, a shopowner would want to make just one fulfillment per order, but there are times when the availability or size of items in the order make it necessary to make more than one shipment, which is why an order can have one or more fulfillments.

What can you do with fulfillments using the Shopify API?

The Shopify API lets you do the following with fulfillments:

  • Get information about one or more fulfillments for an order.
  • Count all or specific fulfillments for an order.
  • Fulfill items in an order.
  • Modify a fulfillment.
  • Cancel a fulfillment.

Fulfillment properties

Name Description
created_at The date and time when the fulfillment was created. The API returns this value in ISO 8601 format.
id The unique numeric identifier for the fulfillment.
line_items
order_id
receipt
status
tracking_company
tracking_number
updated_at The date and time when the fulfillment was last modified. The API returns this value in ISO 8601 format.

Endpoints

GET /admin/orders/order_id/fulfillments.json

Query parameters

By default, requesting a list of fulfillments returns all the fulfillments for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
created_at_max The upper bound on the date and time of creation (its created_at value) for the fulfillments you want to retrieve. Use this parameter to retrieve only fulfillments that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the fulfillments you want to retrieve. Use this parameter to retrieve only fulfillments that were created after a given date and time.
fields A comma-separated list of the properties you want returned for each fulfillment in the result list. By default, the returned list of fulfillments is a list of fulfillments with all the properties included. Use this parameter to restrict the returned list of fulfillments to showing only those properties you specify, with each property name separated by a comma.
limit The maximum number of fulfillments that should be returned. By default, the maximum number of fulfillments returned is 50. Use this parameter to change the maximum number of returned fulfillments from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the fulfillments in a shop.
since_id The lower bound on a fulfillment’s id. Use this parameter to restrict the returned list to only fulfillments whose id is greater than the specified id.
updated_at_max The upper bound on the date and time of a fulfillment’s last update (its updated_at value) for the fulfillments you want to retrieve. Use this parameter to retrieve only fulfillments that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a fulfillment's last update (its updated_at value) for the fulfillments you want to retrieve. Use this parameter to retrieve only fulfillments that were updated before a given date and time.

Example: Fulfill a single line ite

There are times when a shopowner gets an order made up of a few items, and only one of the items is in stock. The shopowner could wait until all the items in the oder were in stock and then ship them all at once, but customers like getting their purchases as soon as possible. Many shopowners prefer to fulfill orders as soon as they are able, shipping order items the moment they're available, even if it means making more than one shipment per order.

In this example, we want to fulfill one item in an order with the following properties:

  • The item to be fulfilled: a green 8GB iPod Nano
  • The id of the order: 450789469
  • The id of the line item in the order: 466157049

To do this, we POST to the order and provide the line item's id in the request body.

Request:

POST /admin/orders/450789469/fulfillments.json

Request body:

{
  "fulfillment": {
    "tracking_number": null,
    "line_items": [
      {
        "id": 466157049
      }
    ]
  }
}

Example: Fulfill all line items at once and notify the customer

The ideal situation for a shopowner is when an order comes in and all the items in the order are in stock. That means the order can be fulfilled with a single fulfillment (that is, one single shipment).

You can fulfill all the line items in an order by specifying all their ids in the request body, but the Shopify API has a convenient shortcut. If you omit the line item ids, all the items in the order will be fulfilled.

In order to notify the customer via email that the order has been fulfilled, we'll use the notify_customer property in the request body and set its value to true.

In this example, we'll fulfill the entire order whose id is 450789469 and notify the customer via email that the order has been fulfilled.

Request:

POST /admin/orders/450789469/fulfillments.json

Request body:

{
  "fulfillment": {
    "tracking_number": "123456789",
    "notify_customer": true
  }
}
