--- layout: resource title: Page intro: "In addition to an online storefront, Shopify shops come with a web page creation tool, allowing a shop to have one or more pages. Shopowners are encouraged to use pages for information that customers will use often, such as an 'About Us' page, a 'Contact Us' page, a page with customer testimonials and so on." ---

Pages are meant to be used for static content rather than as a newsletter or blog. If the shop needs a blog, we recommend that you use the blog feature instead.

What can you do with pages using the Shopify API?

The Shopify API lets you do the following with pages:

  • Get information about one or more of the shop's pages.
  • Count all or specific pages.
  • Create a new page.
  • Modify a page.
  • Delete a page.

Page properties

Name Description
author The name of the author of the page.
body_html Text content of the page, complete with HTML markup.
created_at The date and time when the page was created. The API returns this value in ISO 8601 format.
handle
id The unique numeric identifier for the page.
published_at

This can have two different types of values, depending on whether the page has been published (i.e., made visible to the blog's readers).

  • If the page is published, this value is the date and time when it was published. The API returns this value in ISO 8601 format.
  • If the page is a hidden, this value is null. Changing an page's status from published to hidden changes its published_at property to null.
shop_id The id of the shop to which the page belongs.
template_suffix The title of the page.
title The title of the page.
updated_at The date and time when the page was last modified. The API returns this value in ISO 8601 format.

Endpoints

GET /admin/pages.json

Query parameters

By default, requesting a list of pages returns all the pages for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
created_at_max The upper bound on the date and time of creation (its created_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were created after a given date and time.
fields A comma-separated list of the properties you want returned for each page in the result list. By default, the returned list of pages is a list of pages with all the properties included. Use this parameter to restrict the returned list of pages to showing only those properties you specify, with each property name separated by a comma.
handle
limit The maximum number of pages that should be returned. By default, the maximum number of pages returned is 50. Use this parameter to change the maximum number of returned pages from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the pages in a shop.
published_at_max The upper bound on the date and time of when a page was published (its published_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were published before a given date and time.
published_at_min The lower bound on the date and time of when a page was published (its published_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were published after a given date and time.
published_status

The status – published, unpublished or both – of the pages you want to retrieve. Valid values are:

  • any (default): Use this value to retrieve both published and unpublished pages. This is the default value.
  • published: Use this value to retrieve only pages that are published (created and visible to customers).
  • unpublished: Use this value to retrieve only pages that are unpublished (created, but not yet visible to customers).
since_id The lower bound on a page's id. Use this parameter to restrict the returned list to only pages whose id is greater than the specified id.
title
updated_at_max The upper bound on the date and time of a page's last update (its updated_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a page's last update (its updated_at value) for the pages you want to retrieve. Use this parameter to retrieve only pages that were updated after a given date and time.

GET /admin/pages/count.json

Query parameters

Name Description
created_at_max The upper bound on the date and time of creation (its created_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were created after a given date and time.
published_at_max The upper bound on the date and time of when a product was published (its published_at value) for the products you want to count. Use this parameter to count only products that were published before a given date and time.
published_at_min The lower bound on the date and time of when a product was published (its published_at value) for the products you want to count. Use this parameter to count only products that were published after a given date and time.
updated_at_max The upper bound on the date and time of a product's last update (its updated_at value) for the products you want to count. Use this parameter to count only products that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a product's last update (its updated_at value) for the products you want to count. Use this parameter to count only products that were updated before a given date and time.

Example: Count all the pages

Request:

GET /admin/pages/count.json

Response:

HTTP/1.1 200 OK[]
{
  "count": 3
}
POST /admin/page.json

Example: Create a new page

In this example, we want to create a new page with the title "New Page" and the content "This is a new page."

Request:

POST /admin/pages.json

Request body:

{
  "page": {
    "title": "New Page",
    "body_html": "This is a new page."
  }
}

Response:

HTTP/1.1 201 Created[]
 {
   "page": {
       "author": "development shop",
       "body_html": "This is a new page.",
       "created_at": "2012-04-03T01:45:47-04:00",
       "handle": "new-page",
       "id": 7971183,
       "published_at": "2012-04-03T01:45:47-04:00",
       "shop_id": 1127572,
       "template_suffix": null,
       "title": "New Page",
       "updated_at": "2012-04-03T01:45:47-04:00"
   }
}
