--- layout: resource title: event intro: "Events are generated by specific Shopify resources when specific things happen, such as the creation of an article, the placement or fulfillment of an order, the addition or deletion of a product, and so on. By requesting events, your app can get a 'log' of important occurrences in the operation of a shop." ---

The following Shopify resources generate events:

  • Articles
  • Blogs
  • Collections (both custom collections and smart collections)
  • Comments
  • Orders
  • Pages
  • Products

What can you do with events using the Shopify API?

The Shopify API lets you do the following with events:

  • Get information about one or more events
  • Count all or specific events

Event Properties

Name Description
body
created_at The date and time when the event was created. The API returns this value in ISO 8601 format.
id The unique numeric identifier for the event.
message
subject_id The id of the resource that generated the event.
subject_type The type of the resource that generated the event. This will be one of the following:
  • Article
  • Blog
  • Collection
  • Comment
  • Order
  • Page
  • Product
verb The type of event that took place. Different resources generate different types of event; see Resources and their event verbs and messages (below) for details.

Resources and their event verbs and messages

Each of the event-generating resources generates it own specific set of events, which are listed by resource below.

Article events

Article events mark important points in the lifecycle of an article, from creation to editing to publication to deletion. Blogs and comments generate their own events.

Verb Message Description
create New article created: ''article_url''. The article was created.
destroy Article ''article_url'' was deleted. The article was deleted.
published Article ''article_url'' was published. The article was published.
unpublished Article ''article_url'' was hidden. The article was turned into a draft.
update Article ''article_url'' was updated. The article was updated.

Blog events

Blog events are generated whenever a blog is created, deleted or when its properties are changed. Blog articles and comments generate their own events.

Verb Message Description
create New blog: ''new_blog_url''. The blog was created.
destroy Blog ''blog_name'' was deleted. The blog was deleted.
update Blog ''blog_name'' was updated. The blog was updated.

Collection events

Collection events apply to both custom collections and smart collections. They mark important points in their lifecycles, from creation to changes to deletion.

Verb Message Description
create New collection: ''collection_url''. The collection was created.
destroy Collection ''collection_name'' was deleted. The collection was deleted.
published Collection ''collection_url'' was published. The collection was publsihed.
unpublished Collection ''collection_url'' was hidden. The collection was hidden.
update Collection ''collection_url'' was updated. The collection was updated.

Comment events

A comment event is generated every time a comment to an article is posted. Blogs and their articles generate their own events.

Verb Message Description
create New comment for <a href="''article_url''>''article_title''</a>: ''comment_excerpt'' The comment was created.

Order events

There are a lot of events associated with orders. A few are generated by the order itself, and many more are generated by all the things that can happen in the time between the placing of an order and its closing or cancellation. Order events can be divided into the following categories:

  • Authorization: success, failure and pending
  • Capture: success, failure and pending
  • Email: for confirmation of the order, shipping and cancellation
  • Fulfillment: success, failure, pending, cancellation, restocking and update
  • Order: placing, confirmation, closing, reopening and cancellation
  • Refund: success, failure and pending
  • Sale: success, failure and pending
  • Void: success, failure and pending

Page events

Page events mark important points in the lifecycle of a page, from creation to editing to publication to deletion.

Verb Message Description
create New page: ''page_title''. The page was created.
destroy Page ''page_name''' was deleted. The page was deleted.
published Page ''page_title'' was published. The page was published.
unpublished Page ''page_title'' was hidden. The page was hidden.
update Page ''page_title'' was updated. The page was updated.

Product events

Product events mark important points in the lifecycle of a product, from creation to editing to publication to deletion.

Verb Message Description
create New product: ''product_title''. The product was created.
destroy Product ''product_name'' was deleted. The product was deleted.
published Product "''product_title''" was published. The product was published.
unpublished Product "''product_title''" was hidden. The product was hidden.
update Product "''product_title''" was updated. The product was updated.

Endpoints

GET /admin/events.json

Query parameters

By default, requesting a list of events returns all the events for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
created_at_max The upper bound on the date and time of creation (its created_at value) for the events you want to retrieve. Use this parameter to retrieve only events that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the events you want to retrieve. Use this parameter to retrieve only events that were created after a given date and time.
fields A comma-separated list of the properties you want returned for each event in the result list. By default, the returned list of events is a list of events with all the properties included. Use this parameter to restrict the returned list of events to showing only those properties you specify, with each property name separated by a comma.
filter
limit The maximum number of events that should be returned. By default, the maximum number of events returned is 50. Use this parameter to change the maximum number of returned events from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the events in a shop.
since_id The lower bound on a article’s id. Use this parameter to restrict the returned list to only events whose id is greater than the specified id.

/admin/events/count.json

Example: Count all events

In this example we want to count all the events.

Request:

GET /admin/events/count.json

Response:

HTTP/1.1 200 OK[]
{
  "count": 116
}

Example: Count specific events, filtered by criteria

In this example, we want to count only those events generated on April 18, 2012, eastern daylight time (GMT -4).

Request:

GET /admin/events/count.json?created_at_min=2012-04-18T00:00:00-04:00&created_at_max=2012-04-18T23:59:59-04:00

Response:

HTTP/1.1 200 OK[]
{
  "count": 5
}
