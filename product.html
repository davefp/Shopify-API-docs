--- layout: resource title: Product intro: "A product is an individual item for sale in a Shopify shop. Products are often physical, but don't have to be; a digital download (such as a movie, music or ebook file) also qualifies as a product, as do services (such as equipment rental, work for hire, customization of another product or an extended warranty). Simply put: if it's something for sale in a shop, it's a Product." ---

What can you do with products using the Shopify API?

The Shopify API lets you do the following with ***:

  • Get information about one or more products
  • Count all or specific products
  • Create a new product
  • Modify a product
  • Delete a product

Just about anything that can be done with an product through the shop's admin page can also be done through the API.

Product properties

Name Description
body_html The description of the product, complete with HTML formatting.
created_at The date and time when the product was created. The API returns this value in ISO 8601 format.
handle A human-friendly unique string for the Product automatically generated from its title. They are used by the Liquid templating language to refer to objects.
id The unique numeric identifier for the product. Product ids are unique across the entire Shopify system; no two products will have the same id, even if they're from different shops.
images A list of image objects, each one representing an image associated with the product.
options Custom product property names like “Size”, “Color”, and “Material”. Product variants are based on permutations of these options. A product may have a maximum of 3 options.
product_type A categorization that a product can be tagged with, commonly used for filtering and searching.
published_at The date and time when the product was published. The API returns this value in ISO 8601 format.
tags A categorization that a product can be tagged with, commonly used for filtering and searching.
template_suffix Specifies a special template to be applied only to this product.
title The name of the product. In a shop's catalog, clicking on a product's title takes you to that product's page. On a product's page, the product's title typically appears in a large font.
updated_at The date and time when the product was last modified. The API returns this value in ISO 8601 format.
user_id A unique numeric identifier for the author of the product.
variants A list of variant objects, each one representing a slightly different version of the product. For example, if a product comes in different sizes and colors, each size and color permutation (such as "small black", "medium black", "large blue"), would be a variant.
vendor The name of the vendor of the product.

Endpoints

GET /admin/products.json

Query parameters

By default, requesting a list of products returns all the products for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
collection_id The id of a collection of products that you want to retrieve. Use this parameter to retrieve only products that belong to a specific collection.
created_at_max The upper bound on the date and time of creation (its created_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were created after a given date and time.
fields A comma-separated list of the properties you want returned for each product in the result list. By default, the returned list of products is a list of products with all the properties included. Use this parameter to restrict the returned list of products to showing only those properties you specify, with each property separated by a comma.
handle A unique string for the Product automatically generated from its title used by the Liquid templating language to refer to objects. Use this parameter to retrieve only the product with a specific handle.
limit The maximum number of products that should be returned. By default, the maximum number of products returned is 50. Use this parameter to change the maximum number of returned products from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the products in a shop.
product_type The type of the products to retrieve. Use this parameter to retrieve only products of a specific product type.
published_at_max The upper bound on the date and time of when a product was published (its published_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were published before a given date and time.
published_at_min The lower bound on the date and time of when a product was published (its published_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were published after a given date and time.
published_status The status – published, unpublished or both – of the products you want to retrieve. Valid values are:
  • any (default): Use this value to retrieve both published and draft products.
  • published: Use this value to retrieve only products that are published.
  • unpublished: Use this value to retrieve only products that saved as drafts.
since_id The lower bound on a product’s id. Use this parameter to restrict the returned list to only products whose id is greater than the specified id.
updated_at_max The upper bound on the date and time of a product’s last update (its updated_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a product's last update (its updated_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were updated before a given date and time.
vendor The vendor of the products to retrieve. Use this parameter to retrieve only the products of a specific vendor.

Example: Get a list of products, limiting the number of properties returned

By default, the resulting list of products includes every property for each product in that list. This can be a large amount of data and is often more information than required. The fields input parameter is useful for paring down the properties returned with each product.

In this example, we want to retrieve the list of products in the shop, but want only the id and title of each product.

Request:

GET /admin/products.json?fields=id,title

GET /admin/products/count.json

Query parameters

Name Description
collection_id The id of a collection whose products you want to count. Use this parameter to count only products that belong to a specific collection.
created_at_max The upper bound on the date and time of creation (its created_at value) for the products you want to count. Use this parameter to count only products that were created before a given date and time.
created_at_min The lower bound on the date and time of creation (its created_at value) for the products you want to retrieve. Use this parameter to retrieve only products that were created after a given date and time.
product_type The type of the products to count. Use this parameter to count only products of a specific product type.
published_at_max The upper bound on the date and time of when a product was published (its published_at value) for the products you want to count. Use this parameter to count only products that were published before a given date and time.
published_at_min The lower bound on the date and time of when a product was published (its published_at value) for the products you want to count. Use this parameter to count only products that were published after a given date and time.
published_status

The status – published, unpublished or both – of the products you want to count. Valid values are:

  • any (default): Use this value to count both published and unpublished products. This is the default value.
  • published: Use this value to count only products that are published (created and visible to customers).
  • unpublished: Use this value to count only products that are unpublished (created, but not yet visible to customers).
updated_at_max The upper bound on the date and time of a product’s last update (its updated_at value) for the products you want to count. Use this parameter to count only products that were updated before a given date and time.
updated_at_min The lower bound on the date and time of a product's last update (its updated_at value) for the products you want to count. Use this parameter to count only products that were updated before a given date and time.
vendor The vendor of the products to count. Use this parameter to count only the products of a specific vendor.
