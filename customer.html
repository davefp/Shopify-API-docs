--- layout: resource title: Customer intro: A customer resource instance represents a customer account with the shop. Customer accounts store contact information for the customer, saving logged-in customers the trouble of having to provide it at every checkout. They don't store credit card information; customers will always have to provide this. They also store some additional information about the customer for the benefit of the shopowner, such as the number of orders, the amount of money s/he has spent and number of orders s/he has made throughout his/her history with the shop as well as the shopowner's notes and tags for the customer. ---

The shop's use of customer accounts will depend on its Customer Checkout settings (located on the Checkout and Payment page, accessible via the Preferences menu). There are three options for this setting:

  • Guest checkout only: Customer accounts are disabled. meaning that customers can't log in and can only check out as guests.
  • Guest checkout with optional sign-in: Customer accounts are optional. Customers have the choice of either signing into their account or simply checking out as a guest. Under this setting, customers can create accounts for themselves; the shopowner can also an account for a customer and then invite him/her by email to use it.
  • Sign-in required: Customer accounts are required. Customers can't check out unless logged in, and under this setting, the shopowner must create accounts for them first.

What can you do with customers using the Shopify API?

The Shopify API lets you do the following with customers:

  • Get information about one or more of the shop's customer accounts.
  • Count all or specific customer accounts.
  • Create a new customer account.
  • Modify a customer account.
  • Delete a customer account.

Customer properties

Name Description
accepts_marketing Indicates whether the customer has consented to be sent marketing material via email. Valid values are true and false.
addresses

A list of addresses for the customer. Each address has the following properties:

  • address1: The customer's mailing address.
  • address2: An additional field for the customer's mailing address.
  • city: The customer's city.
  • company: The customer's company.
  • country: The customer's country.
  • country_code: The two-letter country code corresponding to the customer's country.
  • default: Indicates whether this address is the default address for the customer. Valid values are true and false.
  • first_name: The customer's first name.
  • id: A unique numeric identifier for the address.
  • last_name: The customer's last name.
  • name: The customer's name.
  • phone: The customer's phone number.
  • province: The customer's province or state.
  • province_code: The two-letter province or state code for the customer's province or state.
  • zip: The customer's zip or postal code.
created_at The date and time when the customer record was created. The API returns this value in ISO 8601 format.
email The customer's email address.
first_name The customer's first name.
id The unique numeric identifier for the customer.
last_name The customer's last name.
note A note about the customer.
orders_count The number of orders associated with this customer.
state
tags
total_spent The total amount of money that the customer has spent at the shop.
updated_at The date and time when the customer account was last modified. The API returns this value in ISO 8601 format.

Endpoints

GET /admin/customers.json

Query parameters

By default, requesting a list of customers returns all the customers for that shop. You can use the optional query parameters listed below to filter the returned list by specific criteria.

Name Description
fields A comma-separated list of the properties you want returned for each customer in the result list. By default, the returned list of customers is a list of customers with all the properties included. Use this parameter to restrict the returned list of customers to showing only those properties you specify, with each property name separated by a comma.
limit The maximum number of customers that should be returned. By default, the maximum number of customers returned is 50. Use this parameter to change the maximum number of returned customers from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
page The page of the result list to retrieve. Use this in tandem with limit to page through the customers in a shop.
since_id The lower bound on a customer’s id. Use this parameter to restrict the returned list to only customers whose id is greater than the specified id.

GET /admin/customers/search.json

Query parameters

Name Description
f Short for "filters". Use this parameter to filter results.
fields A comma-separated list of the properties you want returned for each customer in the result list. By default, the returned list of customers is a list of customers with all the properties included. Use this parameter to restrict the returned list of customers to showing only those properties you specify, with each property name separated by a comma.
limit The maximum number of customers that should be returned. By default, the maximum number of customers returned is 50. Use this parameter to change the maximum number of returned customers from 1 to 250. Setting this parameter outside the 1 - 250 range will result in an error.
order The field and direction by which results should be ordered. The default value is last_order_date DESC
page The page of the result list to retrieve. Use this in tandem with limit to page through the customers in a shop.
q Short for "query". Use this parameter to search for customers by name.
since_id The lower bound on a customer’s id. Use this parameter to restrict the returned list to only customers whose id is greater than the specified id.
